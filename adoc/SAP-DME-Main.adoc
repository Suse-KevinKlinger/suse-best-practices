= SAP {dme} on SUSE

== Introduction

This guide describes how to prepare your infrastructure for the installation of {dme} on {rke} using {rancher}.
It will guide you through the steps of:

* Installing {rancher}
* Setting up {rke} clusters
* Deploying mandatory components for {dme}

=== Logging in to {rac}

To access the {rac}, you need to log in. You can do this using the console and Helm client.
The easiest way to do so is to use the built-in shell in {rancher}. To access it, navigate to your cluster and click *Kubectl Shell* as shown below:

image::EIC-Rancher-Kubectl-Button.png[title=Rancher Shell Access,scaledwidth=99%]

A shell will open as shown in the image below:

image::EIC-Rancher-Kubectl-Shell.png[title=Rancher Shell Overview,scaledwidth=99%]


You must log in to {rac}. This can be done as follows:

[source, bash]
----
helm registry login dp.apps.rancher.io/charts -u <yourUser> -p <your-token>
----

==== Updating Helm values for the Proxy Cache

To route image requests through your newly created {reg} Proxy Cache (configured in <<prepareRegistryProject>>), you must update the SAP {dme} Helm `values.yaml` file.

Locate the `global.dockerRegistry` parameter in your custom `values.yaml` file and update it to point to your {reg} FQDN and the specific proxy cache project name.

Modify your `values.yaml` file as follows:

[source, yaml]
----
global:
  dockerRegistry: "<PRIVATE_REGISTRY_FQDN>/<PROXY_PROJECT_NAME>"
  # Example: "suse-registry.example.com/proxy-cache-project"
  security:
    imageCredentials:
      username: "<SPR_USERNAME>"
      password: "<SPR_PASSWORD>"
----

NOTE: Ensure that the username and password provided in the `imageCredentials` block correspond to a registry account that has at least read permissions (`Pull` role) assigned within the specified proxy project.
