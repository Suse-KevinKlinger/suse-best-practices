[#istioInstall]
== Installing {istio}

Even though {istio} is available for deployment using the {rancher} Apps, we recommend to use the {rac}.
The {istio} chart can be found at https://apps.rancher.io/applications/istio.

// IMPORTANT::
// SUSE does not offer application support for {istio}.
// For support requests, contact link:https://istio.io/[Istio].

// TODO: when available, add information from SAP regarding to shared cluster.

// TODO: when available, double-check that link includes istio as a requirement and
//       add additional links if necessary
Before deploying {istio}, ensure that the requirements described at https://me.sap.com/notes/3247839 are met.

To deploy the chart, create the related namespace and *imagePullSecret* first.

To create the namespace, run:

[source, bash, subs="attributes"]
----
kubectl create namespace {istio_namespace}
----

[#istioIPS]
Instructions how to create the *imagePullSecret* can be found in <<imagePullSecret>>.

[#istioLIR]
Before you can install the application, you need to log in to the registry.
You can find the instructions in <<LoginApplicationCollection>>.

NOTE: The *{istio_namespace}* namespace, *imagePullSecret*, and registry login established for {istio} will be reused by:
{kiali} at <<kialiInstall>>, {prometheus} at <<prometheusInstall>>, and {grafana} at <<grafanaInstall>>. 

== Deploying the chart

Create a file _values.yaml_ to store some configurations for the {istio} Helm chart.
The configuration might look like the following:

[source, yaml]
----
global:
  imagePullSecrets:
    - name: application-collection
----

To install the application, run:
[source, bash, subs="attributes"]
----
helm install istiod oci://dp.apps.rancher.io/charts/istio \
-f values.yaml \
--namespace={istio_namespace} \
--version {istio_chart_version}
----

For more information on installing and configuring {istio}, check the reference guide at https://docs.apps.rancher.io/reference-guides/istio/.

You can do this by running the following command:

[source, yaml]
----
GRAFANA_PASSWORD=$(kubectl get secret --namespace istio-system grafana -o jsonpath="{.data.admin-password}" | base64 -d ; echo)
cat <<EOF > values.yaml
external_services:
  grafana:
    auth:
      type: basic
      username: admin
      password: $GRAFANA_PASSWORD
    enabled: true
    external_url: http://grafana.istio-system
    internal_url: http://grafana.istio-system
  prometheus:
    custom_metrics_url: http://prometheus-server.istio-system
    url: http://prometheus-server.istio-system
global:
  imagePullSecrets:
    - application-collection
# Optional: Expose the Kiali with an Ingress
# deployment:
#   ingress:
#     enabled: true
EOF
----

To install the application, run:
[source, bash, subs="attributes"]
----
helm install kiali oci://dp.apps.rancher.io/charts/kiali \
-f values.yaml \
--namespace={kiali_namespace} \
--version {kiali_chart_version}
----

NOTE: For more information on installing and configuring {kiali}, check the reference guide at https://docs.apps.rancher.io/reference-guides/kiali/.

[#raNamespaceConfig]
== Configuring the cluster for Restricted Access

To install {elm} and deploy {eic} with **Restricted Access**, the process is now divided into two phases to adhere to the principle of least privilege:

1. **ELM Initialization Phase:** Configuration of the `edgelm` namespace and base cluster permissions.
2. **EIC Deployment Phase:** Configuration of solution-specific namespaces when the deployment reaches the "Prepared" state.

Refer to https://me.sap.com/notes/3618713 for detailed instructions on the restricted resources process via the {elm} UI.

=== Phase 1: ELM Initialization

When you add a new Edge Node with the "Restricted Access cluster" option enabled in the {elm} UI, you will be prompted to download the first `resources.zip` bundle. 

Extract the downloaded ZIP file to a new directory (e.g., `elm-resources`).

Apply the manifests to your cluster by pointing `kubectl` to the extracted directory:

[source, bash]
----
kubectl apply -f elm-resources/
----

This action creates the `edgelm` namespace and applies the foundational security and configuration policies required by {elm}.

[#elmIPS]
Since the `edgelm` namespace is targeted for Istio sidecar injection, it requires an *imagePullSecret* to enable image pulling. For detailed instructions on creating the credentials, refer to <<imagePullSecret>>.

Run the following command to create the secret in the `edgelm` namespace:

[source, bash]
----
kubectl create secret docker-registry application-collection -n edgelm \
  --docker-server=dp.apps.rancher.io \
  --docker-username=<yourUser> \
  --docker-password=<yourPassword>
----

==== Creating the restricted user in Rancher

This restricted user will be utilized by {elm} to securely connect and deploy solutions to your cluster without requiring full administrator privileges.

Before patching the permissions, create the restricted user in Rancher:

1. Navigate to **Users & Authentication** and click *Create*.
2. Set a username (e.g., `eic-restricted`), a password, and select **Global Permissions > Standard User**.
3. **Record this password** and the **User ID** (formatted as "u-XXXXX") for the subsequent configuration steps.

Next, assign this user to your downstream cluster:

1. Navigate to **Cluster > Cluster and Project Members** in the Rancher UI.
2. Click **Add**.
3. Select the restricted user and choose **Cluster Permission > Custom > View Nodes**.

image::SAP-Rancher-Main-Users.png[title=Rancher users list,scaledwidth=99%]

[#raUserConfigPhase1]
==== Patching ELM restricted user permissions

You must now associate your restricted user with the security policies created from the zip file. This grants the user the exact access rights required by {elm}. 

Create a file named `patch-elm-rbac.sh` and add the following script:

[source, bash]
----
#!/bin/bash
set -eu

if [ "$#" -ne 1 ]; then
  echo "Usage: $0 <USER_ID>"
  exit 1
fi
USER_ID="$1"
PATCH_PAYLOAD=$(printf '[{"op":"add","path":"/subjects/-","value":{"apiGroup":"rbac.authorization.k8s.io","kind":"User","name":"%s"}}]' "$USER_ID")

echo "Adding ELM RoleBindings and ClusterRoleBindings for user: $USER_ID"

kubectl patch rolebindings rb-edgelm-manage -n edgelm --type=json -p "$PATCH_PAYLOAD"
kubectl patch rolebindings rb-edgelm-admin -n edgelm --type=json -p "$PATCH_PAYLOAD"
kubectl patch clusterrolebindings crb-edgelm-cluster-admin -n edgelm --type=json -p "$PATCH_PAYLOAD"

echo "ELM patching complete."
----

Make the script executable and run it, replacing `<USER_ID>` with the actual ID (e.g., `u-XXXXX`) you recorded previously:

[source, bash]
----
chmod +x patch-elm-rbac.sh
./patch-elm-rbac.sh <USER_ID>
----

==== Download the restricted access kubeconfig file

1. Log in to the Rancher UI **using the credentials of the restricted user** you created in the previous steps.
2. Navigate to the target cluster.
3. Download the kubeconfig file and use it in the {elm} UI to finish adding the Edge Node.

image::SAP-Rancher-Main-Download.png[title=Rancher download kubeconfig,scaledwidth=99%]

=== Phase 2: EIC Deployment

Once the Edge Node is available, start the {eic} deployment from the {elm} UI. The wizard will eventually pause, and the operation will transition to a **Prepared** state. 

At this point, {elm} provides a second zip file (typically named `restricted-resources-Edge_Integration_Cell-<version>.zip`) containing the specific manifests for the EIC solution. 

Extract the contents of this new zip file to a directory (for example, `eic-resources`) and apply them to your cluster:

[source, bash]
----
kubectl apply -f eic-resources/
----

This will create the remaining namespaces: `istio-gateways`, `edge-icell`, `edge-icell-services`, `edge-icell-secrets`, and `edge-icell-ela`.

==== Configuring ImagePullSecrets for EIC namespaces

As described in <<imagePullSecret>>, all newly created namespaces (except `edge-icell-secrets`) have the Istio injection label and require the *imagePullSecret*. 

To avoid running the creation command manually for each namespace, you can use the following script to automate the process. Create a file named `create-eic-ips.sh` and add the following content:

[source, bash]
----
#!/bin/bash
NAMESPACES_INJECTION=("istio-gateways" "edge-icell" "edge-icell-services" "edge-icell-ela")
for ns in "${NAMESPACES_INJECTION[@]}"; do
  echo "Creating imagePullSecret for namespace: $ns"
  kubectl create secret docker-registry application-collection -n $ns \
    --docker-server=dp.apps.rancher.io \
    --docker-username=<yourUser> \
    --docker-password=<yourPassword>
done
echo "ImagePullSecrets deployment complete."
----

Remember to replace `<yourUser>` and `<yourPassword>` with your actual registry credentials inside the file before executing it:

[source, bash]
----
chmod +x create-eic-ips.sh
./create-eic-ips.sh
----

[#raUserConfigPhase2]
==== Patching EIC restricted user permissions

Similarly, you must associate your restricted user with the new solution-specific security policies applied from the second zip file.

Create a file named `patch-eic-rbac.sh` and add the following script:

[source, bash]
----
#!/bin/bash
set -eu

if [ "$#" -ne 1 ]; then
  echo "Usage: $0 <USER_ID>"
  exit 1
fi
USER_ID="$1"

echo "Adding EIC RoleBindings for user: $USER_ID"

PATCH_PAYLOAD=$(printf '[{"op":"add","path":"/subjects/-","value":{"apiGroup":"rbac.authorization.k8s.io","kind":"User","name":"%s"}}]' "$USER_ID")

for RB in \
  "istio-gateways:rb-edgelm-manage" \
  "istio-gateways:rb-istio-gateways-admin" \
  "edge-icell-services:rb-edgelm-manage" \
  "edge-icell-services:rb-admin" \
  "edge-icell-services:rb-app-admin" \
  "edge-icell:rb-edgelm-manage" \
  "edge-icell:rb-admin" \
  "edge-icell:rb-app-admin" \
  "edge-icell-secrets:rb-edgelm-manage" \
  "edge-icell-secrets:rb-admin" \
  "edge-icell-ela:rb-edgelm-manage" \
  "edge-icell-ela:rb-admin" \
  "edge-icell-ela:rb-app-admin"
do
  NAMESPACE="${RB%%:*}"
  NAME="${RB##*:}"
  echo "Patching $NAME in namespace $NAMESPACE..."
  kubectl patch rolebindings "$NAME" -n "$NAMESPACE" --type=json -p "$PATCH_PAYLOAD"
done

echo "EIC patching complete."
----

Make it executable and run it with your `USER_ID`:

[source, bash]
----
chmod +x patch-eic-rbac.sh
./patch-eic-rbac.sh <USER_ID>
----

Once the scripts have executed successfully, return to the {elm} UI and click **Resume deployment** to finalize the {eic} installation.
